Для разработки шаблонов проекта «Час ЕГЭ» предпочтительна среда Linux из-за использования символических ссылок, но для выполнения готовых заданий на сайте она не обязательна

Вместо того чтобы устанавливать операционную систему Linux напрямую, мы выбрали более гибкий и безопасный подход — использование виртуализации. Для этого мы скачали и установили программу VirtualBox, которая позволяет запускать гостевые ОС в изолированной среде поверх уже работающей системы. Главное преимущество такого решения — высокая степень изоляции: вредоносные эксперименты внутри виртуальной машины не могут повредить хостовую систему, а с помощью снимков состояния (snapshots) легко откатить изменения в случае ошибки. Однако у этого подхода есть и недостаток: виртуальная машина потребляет дополнительные ресурсы (процессор, оперативную память, дисковое пространство), что создаёт заметную нагрузку на компьютер, особенно при выполнении ресурсоёмких задач. Ниже приводится пошаговое описание процесса установки и запуска виртуальной машины в среде VirtualBox.

\quad 1.\quad Для установки среды виртуализации необходимо перейти на официальный сайт проекта VirtualBox по адресу https://www.virtualbox.org. На главной странице расположена кнопка «Download». После нажатия открывается страница со списком доступных версий для различных хостовых операционных систем. Мы выбрали пункт «Windows hosts» так как на нашем устройстве установлена Windows 11. После завершения загрузки запускаем установочный файл и следуем инструкциям мастера установки: принимаем лицензионное соглашение, выбираем компоненты, указываем папку для установки и завершаем процесс. После успешной установки VirtualBox становится доступен для создания и настройки виртуальных машин.

\quad 2.\quad Далее, для того чтобы виртуальная машина могла работать, необходимо включить виртуализацию. В нашем случае хостовая система — Windows 11. Чтобы проверить, активирована ли на ней виртуализация, необходимо открыть диспетчер задач. Это можно сделать несколькими способами: нажать правой кнопкой мыши по панели задач (или по кнопке «Пуск») и в появившемся контекстном меню выбрать пункт «Диспетчер задач». Альтернативный вариант — использовать комбинацию клавиш Ctrl + Alt + Del и в открывшемся синем меню выбрать «Диспетчер задач». В окне диспетчера нужно перейти на вкладку «Производительность».

\quad Если в диспетчере задач указано, что виртуализация отключена, требуется зайти в настройки BIOS и активировать соответствующую опцию. Для этого необходимо перезагрузить компьютер. В момент, когда экран только загорается и появляется логотип производителя, нужно нажать определённую клавишу (или комбинацию) для входа в BIOS. На стационарных компьютерах это чаще всего клавиша Delete (Del), реже — F2 или F10. На ноутбуках обычно используется F2 (иногда с зажатой клавишей Fn). Чтобы не пропустить момент, можно сразу после нажатия кнопки перезагрузки начать многократно нажимать нужную клавишу.

\quad После входа в BIOS/UEFI необходимо найти раздел «System Configuration», отвечающий за настройки процессора. Внутри этого раздела нужно найти параметр «Virtualization Technology». Перемещение по меню осуществляется стрелками на клавиатуре, выбор пункта — клавишей Enter. Для включения виртуализации необходимо выбрать значение «Enabled». Затем следует сохранить изменения, нажав клавишу «F10» и выбрав «Yes». После сохранения компьютер перезагрузится, и виртуализация будет активирована. Теперь VirtualBox сможет запускать гостевые операционные системы с полной производительностью.

\quad 3. \quad Ubuntu можно скачать с официального сайта по ссылке: https://ubuntu.com/download/desktop. В нашем случае была использована версия Ubuntu 22.04.3. После этого нужно открыть VirtualBox и нажать кнопку «Создать». При заполнении полей необходимо передвинуть ползунок вправо, чтобы увеличить объём оперативной памяти, выделяемой виртуальной машине. Для стабильной работы Ubuntu минимальный размер памяти должен составлять не менее 2 гигабайт (иначе система просто не запустится). Остальные настройки мы оставили по умолчанию. После создания виртуальной машины мы перешли в её «Настройки», затем в раздел «Носители». Там, напротив строки «Контроллер: IDE», нужно нажать на значок диска с плюсом, чтобы добавить установочный образ Ubuntu.

Необходимо заполнить все нужные поля (раскладка клавиатуры, имя пользователя, пароль) и выбрать параметры установки.
