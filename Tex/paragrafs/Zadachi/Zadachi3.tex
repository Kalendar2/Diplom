В зависимости от этого индекса заполняются массивы \texttt{objectsArray1…objectsArray4} и список единиц измерения \texttt{measurements}.

Массивы сортируются по возрастанию с помощью функции \texttt{.sort((a, b) => a[1] - b[1])}, для последующего выбора объектов. Для типов величины не являющихся площадью или объёмов (typeIndex < 3) значение valueB (самый крупный объект) с определённой вероятностью умножается на соответствующий коэффициент (1000, 24 или 1000). Это нужно, чтобы случайным образом переводить из одной единицы измерения в другую, что повышает разнообразие генерируемых заданий.

Если выбранный маленький объект (sluchIndexSmallObject) имеет индекс в начале списка, где находятся объекты с наименьшими показателями, то с вероятностью 1/2 он заменяется на один из крупных объектов из того же массива \texttt{smallObjects} ближе к концу списка. Аналогично для средних объектов (sluchIndexMediumObject) – они могут быть заменены на более крупные из mediumObjects, а также поменяться местами с большими объектами. Такой способ выбор позволяет снизить вероятность того, что два объекта из одного массива будут близкими по своим значениям.

Эти подмены управляются флагами \texttt{sluchValueS}, \texttt{sluchValueM}, \texttt{turnValueBToValueM} и влияют на то, какие названия попадут в итоговое задание.

Для каждого типа величины определён массив коэффициентов \texttt{toBaseCoeff}, переводящий единицы измерения в базовую (например, для длины: мм→м, см→м, м→м, км→м). Для каждого из четырёх объектов вычисляется его числовое значение в базовой единице.

Полученные четыре числа в базовой единице сортируются по возрастанию. Затем проверяется, что каждое следующее значение больше предыдущего минимум в 5 раз, иначе генерируется ошибка \texttt{throw new Error(...)}, что заставляет \texttt{retryWhileError} повторить генерацию.
После этого идёт ещё одна проверка, на наличие двух одинаковых объектов в задании. Таким образом гарантируется, что величины будут хорошо различимы и задание не будет двусмысленным.
