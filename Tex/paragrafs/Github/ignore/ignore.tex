Для того чтобы сообщить Git, какие файлы или каталоги не нужно отслеживать (например, временные файлы), в корне репозитория создаётся файл \texttt{.gitignore}. В нём перечисляются пути игнорируемых объектов.

Чтобы создать этот файл, необходимо открыть терминал, перейти в каталог репозитория и выполнить команду: \texttt{touch .gitignore}.

Если файл или каталог уже ранее был добавлен командой \texttt{git add} или уже содержится в репозитории, простое добавление правила в \texttt{.gitignore} не остановит его отслеживание. В таком случае нужно сначала удалить файл из индекса Git с помощью команды: \texttt{git rm --cached FILENAME}.

Файл .gitignore является скрытым, поскольку его имя начинается с точки. Обычные команды ls не показывают такие файлы. Чтобы увидеть их, используется ключ -a: \texttt{ls -a}. Для редактирования .gitignore подойдёт любой текстовый редактор. В файл записываются пути к игнорируемым файлам или каталогам — по одному правилу на строку.

После сохранения файла Git автоматически перестанет учитывать указанные объекты при выполнении \texttt{git status}, \texttt{git add .} и других операциях. 

Это не единственный способ указать Git, какие файлы следует игнорировать. Существуют также глобальные и локальные настройки.

Глобальное игнорирование действует для всех репозиториев на данном компьютере. Для этого нужно создать отдельный файл, например, \texttt{~/.config/git/ignore}, и записать в него шаблоны игнорируемых файлов, по одному на строку. Затем этот файл подключается командой: \texttt{git config --global core.excludesfile ~/.config/git/ignore}. Такой способ удобен для исключения системных временных файлов, которые не нужно добавлять ни в один репозиторий.

Локальное игнорирование происходит без создания \texttt{.gitignore}. Оно применяется только к текущему репозиторию и не передаётся другим пользователям. Для этого используется файл \texttt{.git/info/exclude}, который находится в служебной папке \texttt{Git}. Его синтаксис полностью совпадает с \texttt{.gitignore}, но он не включается в репозиторий. Этот способ подходит для персональных файлов, которые не должны попадать в общий репозиторий.
