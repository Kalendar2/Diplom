Создание SSH-ключа для возможности подключиться к удалённому репозиторию происходит в несколько этапов.
Для начала нужно проверить наличие ключа, введя следующие команды:
\newline \verb|cd ~/.ssh|
\newline \texttt{ls}
Если файлов с названиями \texttt{id\_dsa} и \texttt{id\_dsa.pub} нет (открытый и приватный ключ), то можно создать их используя команду:
\newline  \texttt{ssh-keygen -o}
\newline \quad Далее нужно открыть содержимое файла dsa.pub командой:
\newline \verb|cd ~/.ssh/ id_dsa.pub|

Далее добавили свой ключ себе в аккаунт на GitHub, чтобы иметь возможность подключиться к удалённому репозиторию [2]. А также прописали ещё одну команду, необходимую для подключения.

SSH-ключ — это пара файлов (приватный и публичный), которые используются для безопасного подключения к удалённому компьютеру или серверу по протоколу SSH (Secure Shell). Создание SSH-ключа делится на несколько этапов.
В начале следует проверить наличие ключа, введя следующие команды в терминале:
\newline \quad \texttt{cd ~/.ssh}
\newline \quad \texttt{ls}

Если файлов с названиями \verb|id_rsa| (открытый ключ) и \verb|id_rsa.pub| (приватный ключ) нет , то нужно создать их, с помощью команды:
\newline \quad \texttt{ssh-keygen -o}

Затем необходимо открыть содержимое файла \verb|id_rsa.pub| введя команду:
\newline \quad \verb|cat ~/.ssh/id_rsa.pub|

После чего нужно добавить свой ключ в аккаунт на GitHub, чтобы иметь возможность подключиться к удалённому репозиторию [2]. А также прописали ещё одну команду для проверки: \texttt{ssh -T git@github.com}.

Перейдя в репозиторий на GitHub «ЧАС-ЕГЭ» (https://github.com/nickkolok/chas-ege/), нужно нажать кнопку «Fork» в правом верхнем углу. После этого в личном аккаунте GitHub появляется собственная копия репозитория (форк), в которую можно вносить изменения без риска повредить оригинал.

Открыв VirtualBox и войдя в свою виртуальную машину, следует запустить терминал и перейти в папку, куда ранее был склонирован проект «Час ЕГЭ». C помошью команды: \texttt{cd git/chas-ege}.

Перед началом работы необходимо представиться Git, так как каждая фиксация изменений (коммит) подписывается именем и электронной почтой:
\newline \texttt{git config ---global user.name «Фамилия Имя»}
\newline \texttt{git config ---global user.email «электронная почта пользователя»}
\newline Можно также настроить цветной вывод команд Git для лучшей читаемости:
\texttt{git config ---global color.ui true}
\newline Теперь нужно добавить свой форк на GitHub в список удалённых репозиториев (remotes) локального клона:
\texttt{git remote add myfork git@github.com:«GitHubNik»/chas-ege.git}, где «GitHubNik» - ник пользователя на гитхабе. 

После выполнения этой команды можно отправлять изменения в свой форк командой \texttt{git push myfork}. Убедиться, что удалённый репозиторий добавился, можно командой \texttt{git remote -v}.

Основные команды для создания и отправления изменений в удалённый репозиторий:
\\Переключение на основную ветку (devel):
\\ \texttt{git checkout devel}
\\ \quad Обновление текущей ветки из удалённого репозитория (получение изменений без слияния):
\\ \texttt{git fetch origin devel}
\\ \quad Если нужно сразу слить изменения с текущей веткой:
\\ \texttt{git pull origin devel}
\\ \quad Создание новой ветки и переход на неё:
\\ \texttt{git checkout -b newtask-777}
\\ \quad Проверка состояния рабочей директории, какие файлы изменены, какая ветка активна:
\\ \texttt{git status}
\\ \quad Добавление всех изменений в индекс (подготовка к коммиту):
\\ \texttt{git add .}
\\ \quad Фиксация изменений с сообщением:
\\ \texttt{git commit -m «Внесены изменения в файл ...»}
\\ \quad Отправка новой ветки в свой форк (удалённый репозиторий myfork):
\\ \texttt{git push myfork newtask-777:myfork newtask-777:}
\\ \quad Приведение локальных файлов в соответствие с актуальным состоянием удалённой ветки 
\\ \texttt{git pull origin devel}
\\ \quad Просмотр всех удалённых репозиториев:
\\ \texttt{git remote -v}
\\ \quad Слияние изменений из ветки devel в текущую ветку: 
\\ \texttt{git merge devel}
\\ \quad Удаление локальной ветки: 
\\ \texttt{git branch -d newtask-777}
